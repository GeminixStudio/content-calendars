🔧 Setup Guide — 2 steps, ~10 minutes total

Complete both steps below. GitHub Pages gives Dima a permanent shareable URL. Google OAuth enables auto-saving to Google Sheets.

1
Go to github.com and create a free account if you don't have one. Then click New repository.
2
Name the repo anything — e.g. content-calendar. Set it to Public. Check Add a README file. Click Create repository.
3
Click Add file → Upload files. Drag the dima_content_calendar.html file in. Important: rename it to exactly index.html before uploading, or rename it after via the pencil icon. Commit the file.
4
Go to Settings → Pages (left sidebar). Under Branch, select main and click Save.
5
Wait ~60 seconds, then refresh. GitHub will show a green banner with your live URL — it will look like:
https://YOUR-USERNAME.github.io/content-calendar/
That's Dima's permanent shareable link. They can bookmark it or save it to their home screen.
💡 To update the calendar later — just upload a new version of the file to the same repo. The live URL stays the same.

4
Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID. Choose Web application.
Under Authorized JavaScript origins, add your GitHub Pages URL:
